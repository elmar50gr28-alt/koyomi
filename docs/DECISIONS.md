# KOYOMI Decisions

## 2026-07-16

- KOYOMI prioritizes iPhone and smartphone operation.
- Person information is entered once and shared across all divination systems.
- Data required by each divination system must not be deleted.
- Duplicate input must be removed.
- World names, birthplaces, and time zones are first-class requirements.
- Specialist fields belong in detailed settings.
- Input should autosave.
- Duplicate/manual-save/admin buttons are removed from general screens.
- "人物台帳" is renamed to "家族・大切な人" in general UI.
- Bottom navigation hides during input.
- Unnecessary automatic scrolling is forbidden.
- Person profile rebuilds must be done on feature branches.
- `main` is protected as the stable production branch.
- AI must not change divination calculations.
- Mitsunome explains and translates results.
- Existing root documents remain, but `docs/` is the canonical specification set.
