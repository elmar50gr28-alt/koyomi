# KOYOMI Epics

GitHub Issuesで管理するためのEpic一覧です。各Epicは必要に応じてPhase、実装Issue、検証Issue、レビューIssueへ分割します。

##Epic: 四柱推命

- 目的: プロ鑑定に耐える決定論的計算エンジンと知識データベースを完成させる。
- 現在地: Phase 1-4で基礎、旺衰、格局、用神、大運、根拠、品質監査を構築中。
- 完了条件: sourceIds、reviewStatus、命例、回帰テスト、流派差表示が揃う。

##Epic: 奇門遁甲

- 目的: 時間、方位、目的を扱う行動ナビゲーションとして実装する。
- 完了条件: 盤生成、流派設定、根拠、方位解釈、検証命例が揃う。

##Epic: 宿曜

- 目的: 日常で使える相性、日取り、月のリズムを提供する。
- 完了条件: 宿計算、暦連携、根拠、相性導線が揃う。

##Epic: 西洋占星術

- 目的: 生年月日、出生時刻、出生地から出生図と今日の流れを扱う。
- 完了条件: 天体位置、ハウス、アスペクト、トランジット、根拠表示が揃う。

##Epic: 姓名判断

- 目的: 統合プロフィールの氏名情報を再入力なしで使う。
- 完了条件: 字体、かな、カナ、ローマ字、姓名構造、流派差を扱う。

##Epic: AI鑑定

- 目的: 決定論的エンジンの結果を生活語へ翻訳する。
- 完了条件: AIが計算せず、根拠、confidence、注意点を保持して説明する。

##Epic: UI/UX

- 目的: スマホで毎日使える運命ナビゲーションにする。
- 完了条件: iPhone片手操作、初回登録後の本人自動選択、1タップ鑑定導線が揃う。

##Epic: Apple Watch

- 目的: 今日の要点を短く確認できる補助デバイス対応を行う。
- 完了条件: 今日の運勢、暦、注意時間、短い根拠を表示できる。

##Epic: Garmin

- 目的: 軽量、低電力、オフライン前提のウェアラブル対応を検討する。
- 完了条件: 同期項目、表示粒度、データ保持方針が決まる。

##Epic: テスト

- 目的: 既存機能を壊さず、エンジン品質を継続的に確認する。
- 完了条件: 100件以上の命例、静的検証、モバイル回帰、Pages検証が通る。

##Epic: ドキュメント

- 目的: チャット履歴に依存しない仕様管理を行う。
- 完了条件: Vision、Architecture、Roadmap、Status、Rules、Contributingが揃う。
