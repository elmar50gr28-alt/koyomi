# KOYOMI Roadmap

##現在位置

現在は四柱推命知識エンジンのPhase 4品質向上まで進行中です。新しい占術追加より、既存エンジンの根拠、流派差、命例、レビュー状態、テスト密度を優先します。

##Phase 1: 基礎安定化

- 既存アプリの起動安定化。
- localStorage、Service Worker、基本導線の安全化。
- iPhone SafariとPCブラウザで起動確認。

##Phase 2: 四柱推命基礎エンジン

- 十干、十二支、蔵干、十神、十二運を構造化。
- 四柱計算APIと検証基盤を作成。
- GitHub Actionsで静的検証とエンジンテストを実行。

##Phase 3: 旺衰・格局・用神・大運

- 月令、根、透干、五行分布、調候、身旺身弱を構造化。
- 格局候補、従格候補、化格候補を分離。
- 用神メソッド、大運方向、開始時期、流派設定を追加。

##Phase 4: 根拠・命例・品質監査

- 古典資料インデックス、原文、要約、現代解釈を分離。
- ルールごとのsourceIds、confidence、reviewStatusを整理。
- 重複ルール、矛盾候補、低confidence項目を一覧化。
- テストケースを100件以上へ拡充。
- 実装率レポートと`docs/bazi/IMPLEMENTATION_STATUS.md`を更新。

##Phase 5: 四柱推命の精密化

- 24節気の正確な時刻計算。
- 大運開始時期の精密化。
- 流年、月運、日運の構造化。
- 実在命例の採用手順を整備。

##Phase 6: 統合プロフィール完成

- 本人プロフィール、家族・友人、プロモードを安定化。
- personIdで全占術入力を共通化。
- 各占術画面で重複入力をなくす。

##Phase 7: 奇門遁甲

- 盤生成、方位、時間、目的、流派設定を構造化。
- 行動ナビゲーションとして日常導線へ統合。

##Phase 8: 他占術統合

- 宿曜、西洋占星術、姓名判断、数秘、タロット、ルーンを統合プロフィール参照で実装。
- 各占術で根拠、confidence、reviewStatusを保持。

##Phase 9: AI説明レイヤー

- みつのめ姉さんを計算エンジンではなく説明レイヤーとして統合。
- 初心者向け説明と専門家向け根拠を切り替え可能にする。

##Phase 10: ウェアラブル対応

- Apple Watch対応を優先。
- Garmin対応は軽量データ同期と低電力表示を前提に設計。

##Phase 11: 外部テスト公開

- GitHub Pagesまたは同等のHTTPS環境でテスターが確認できる状態にする。
- 既存データを壊さず更新できることを確認する。

##次フェーズ

次はPhase 5として、四柱推命の節気時刻、大運開始、流年・月運、実在命例採用フローを精密化します。
