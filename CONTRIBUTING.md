# Contributing to KOYOMI

##基本方針

KOYOMIは「今日を読み、今日を暮らす」ための術数OSです。開発では速度より品質、見た目より根拠、機能追加より既存機能を壊さないことを優先します。

##AI(Codex)が守ること

- 変更前に影響範囲を確認する。
- 1回の変更は1目的に絞る。
- 既存データ、既存占術、既存UI導線を不用意に削除しない。
- 未実装を実装済みと報告しない。
- 根拠、sourceIds、confidence、reviewStatusを保持する。
- 古典原文、要約、現代解釈、AI生成文を混同しない。
- 流派差を勝手に統一しない。
- 人物情報を重複入力させない。
- mainへ直接コミットしない。
- featureブランチで作業し、PRでmainへ提案する。

##ブランチ運用

- `main`: 公開対象。直接コミット禁止。
- `feature/*`: 機能、エンジン、ドキュメント改修。
- `backup/*`: 破壊的変更前の退避。

##テスト

変更内容に応じて以下を確認します。

- JSONデータがすべてパースできる。
- 四柱推命エンジンテストが通る。
- 静的検証が通る。
- Service Workerのキャッシュ対象に存在しないファイルがない。
- iPhone Safari相当の入力導線を壊していない。

##PR

PRには以下を含めます。

- 変更内容
- 影響範囲
- テスト結果
- 残課題
- mainへmergeしてよいか判断するための注意点
