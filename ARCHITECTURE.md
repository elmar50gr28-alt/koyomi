# KOYOMI Architecture

##システム構成

KOYOMIは静的Webアプリとして構成します。GitHub Pagesで配信でき、PWAとしてオフライン起動できることを前提にします。

- `index.html`: 入口ページ。今日の導線とアプリ本体への入口を持つ。
- `today.html`: 今日の暦を軽量に確認する画面。
- `app.html`: 統合アプリ本体。プロフィール、今日、各占術、設定を扱う。
- `service-worker.js`: 静的ファイル、エンジン、データベースをキャッシュする。
- `manifest.webmanifest`: PWAメタデータを定義する。
- `src/bazi/`: 四柱推命の決定論的エンジン。
- `data/bazi/`: 四柱推命の知識データベース、根拠、テストケース、品質監査。
- `tests/`: 静的検証、四柱推命エンジン検証、モバイル回帰検証。
- `docs/`: 仕様、方針、現状、専門領域別ドキュメント。

##データ構造

人物情報はpersonIdで一元管理します。本人、家族・友人、顧客の区分は保持しますが、各占術画面で氏名、生年月日、出生時刻、出生地を再入力させません。

プロフィールは端末内保存を原則にします。未知フィールドは保持し、将来のデータ移行で失わないようにします。

四柱推命データは以下を分離します。

- 基礎定義: 十干、十二支、五行、陰陽、蔵干、十神、十二運。
- 計算ルール: 暦境界、月令、根、通変、格局、用神、大運。
- 根拠: 古典資料インデックス、原文、要約、現代解釈、sourceIds。
- 品質: confidence、reviewStatus、矛盾候補、重複統合候補。
- 命例: 合成回帰命例、典型命例、境界命例、将来の実在命例。

##エンジン構成

四柱推命エンジンは`src/bazi/index.js`を公開入口とし、計算、強弱、格局、用神、大運、解釈、根拠、検証を分離します。

AIはエンジンではありません。AIは計算済みの構造化結果、根拠、注意点を生活語へ翻訳する説明レイヤーです。

将来の奇門遁甲、宿曜、西洋占星術、姓名判断も同じ原則で構成します。

- 入力は統合プロフィールを参照する。
- 計算は決定論的に行う。
- 根拠と流派差を保持する。
- 初心者向け説明と専門家向け根拠を分離する。

##UI構成

UIはiPhoneファーストです。

- 1画面1目的。
- 主要操作は親指が届く位置に置く。
- 入力中は下部ナビゲーションを隠す。
- Visual Viewport APIとsafe-area-insetに対応する。
- 100vh依存を避け、100dvhまたは実測viewportを使う。
- 横スクロールを禁止する。
- タップ領域は最低44px相当を確保する。

初回は本人プロフィールを登録します。2回目以降は本人を自動選択し、通常鑑定で人物選択を要求しません。

##API構成

内部APIはブラウザ内で完結します。

- `calculateBazi`
- `calculateBaziChart`
- `calculateFourPillars`
- `calculateTenGod`
- `calculateTwelveStage`
- `evaluateBasicStemRelations`
- `evaluateBasicBranchRelations`
- `validateBaziResult`
- `validateBaziPhase2Result`
- `validateBaziPhase3Result`

外部API依存は最小化します。個人情報を外部APIへ送信する設計は禁止します。

##将来連携

Apple Watchは今日の要点、注意時間帯、短い根拠リンクを表示する補助画面として扱います。Garminは軽量データ、低電力、オフライン同期を前提に検討します。
